## Previous round — 2026-07-22: Feed **Drafts** — multiple saved monthly plans per (channel, month) with "Set as final" (verify green, 150 tests)
The Feed page now supports **drafts**: save several alternative monthly arrangements for a channel, edit them freely without touching the live schedule, and commit one when ready.
- **The safe model:** the live schedule (`post.content.date`) is "draft zero". A draft is a pure **overlay** — `{postId → date}` + its own cadence — stored on the brand (`BrandSpace.feedDrafts?: FeedDraft[]`, additive, zero migration). Loading a draft renders/edits the grid from that overlay; **posts and the calendar are untouched** until "Set as final".
- **Set as final** writes the draft's placements onto the real posts via the existing `assignToDate → resolveDateWrites` (so linked planner slots + calendar sync), and — per the user's decision — **unschedules any in-month post the draft omits**, so the live schedule ends up exactly equal to the draft. A confirm shows the counts first ("Schedules N, unschedules M"). Posts are never deleted; only `content.date` changes (reversible).
- **Cross-board by nature:** a draft plans a whole channel-month, so editing one forces "All boards" (the board filter is disabled).
- **UI in `FeedPlannerView`:** a "Drafts" dropdown (Live vs saved plans, with rename/delete + "Save current view as draft"), a burgundy **draft banner** (Editing draft · name · Set as final · Exit to live) that makes it unmistakable you're not editing live, and a small confirm modal. A draft belongs to a specific (channel, month) — changing either drops back to Live.
- **Pure layer:** `services/feedDrafts.ts` (`feedDraftsFor`, `createDraftFromArrangement`, `draftDateResolver`, `writeDraftDate`, `setDraftCadence`, `finalizePlan`) + `buildFeedMonth`/`unplacedItems`/`orderedFeedItems` gained an optional `getDate` resolver so the SAME grid code renders Live or a draft. 8 new tests (150 total).
- ✅ verify 0 errors. **Browser harness passed** (deleted before commit): Save-as-draft captured the live arrangement; unscheduling a tile in the draft changed only the draft (live intact); Set as final committed it (p1 unscheduled, confirm read "Schedules 3, unschedules 1"), returned to Live, kept the draft; switching channel/month reset to Live; TikTok didn't see IG's draft.

## Previous round — 2026-07-22: Cadence remembered per (channel, month), auto-saved on the brand (verify green, 142 tests)
The Feed cadence now persists **independently for every (channel, month)** — July can be daily while August is 2/week, and in the same month TikTok can differ from Instagram — with **no Save button** (auto-saved the instant you change it).
- **Root cause of the old bug:** cadence was stored on the selected board `node.feedPlanner[channel]`, so it applied to *all* months, and in the default "All boards" view there's no node → it wasn't saved at all (vanished on reload).
- **Fix — cadence moved to the brand**, keyed `feedCadence[channel]["YYYY-MM"]` (`BrandSpace`, additive optional field, zero migration). Pure helpers in `services/feedPlanner.ts` (`feedMonthKey`, `resolveMonthCadence`, `writeMonthCadence`) — an untouched month resolves to the neutral default (3/week), **no hidden inheritance** between months, per the user's "months are independent" decision. `hooks/useBrandSpaces.ts` gained `updateBrandFeedCadence` (deep-merges, same anti-clobber rule as `updateBrandProfile` / mistake #14); persistence rides the existing gated+debounced brand save. `FeedPlannerView` reads a single source of truth from the `brandFeedCadence` prop via `resolveMonthCadence` and writes through `onUpdateFeedCadence` — the old local `cadenceState` + node-reading effect are gone.
- **Discoverability by design:** no "Saved" noise — the proof it's remembered is that the stepper shows the right value when you navigate between months/channels (Notion/Figma style).
- ✅ verify 0 errors, 142 tests (5 new for per-month storage). **Browser harness test passed** (deleted before commit): set July IG = every-1-day → August IG resolved to default 3/week → back to July still daily → TikTok July independent → the stored map was exactly `{instagram:{2026-07:daily, 2026-08:5/week}, tiktok:{2026-07:every-3-days}}`.
- **Next round (blueprinted, not built): Drafts** — multiple saved monthly arrangements per (channel, month). A draft = the channel's monthly plan (`{postId→date}` + cadence) on `BrandSpace.feedDrafts?`; the live schedule is "draft zero"; loading a draft renders the grid from its date-map overlay without touching posts; "Set as final" writes via the existing `resolveDateWrites`.
